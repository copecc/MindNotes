# 矩阵

矩阵分组的核心是二维索引管理。常见题型要么在边界上做层次遍历，要么利用首行首列做标记，要么依赖二维单调性进行剪枝。

## 矩阵置零

!!! problem "[73. 矩阵置零](https://leetcode.cn/problems/set-matrix-zeroes/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个 `m x n` 的矩阵 `matrix`，如果某个元素等于 `0`，就需要把它所在的整行和整列都设为 `0`。请直接在原矩阵上完成修改，并返回处理后的矩阵状态。

    题目关注的是如何在不破坏原始信息的前提下完成整行整列清零，通常需要尽量只使用常数额外空间。

如果直接在遍历时修改矩阵，会把“原本需要置零的位置”污染掉，导致信息丢失。更稳妥的做法是把首行和首列当作标记区。

先扫描一遍矩阵，记录首行是否原本就含有 `0`，首列是否原本就含有 `0`。然后对其余位置做标记：一旦发现 `matrix[i][j] == 0`，就把 `matrix[i][0]` 和 `matrix[0][j]` 置为 `0`，表示第 `i` 行和第 `j` 列需要清零。

第二遍再根据标记回填内部区域；最后如果首行或首列本来就含有 `0`，再单独清空首行或首列。这样只使用了常数级额外空间。

## 螺旋矩阵

!!! problem "[54. 螺旋矩阵](https://leetcode.cn/problems/spiral-matrix/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个 `m` 行 `n` 列的矩阵 `matrix`，请按照螺旋顺序返回矩阵中的所有元素。遍历需要从外层向内层一圈一圈展开，直到所有位置都被访问。

    输出顺序固定为螺旋顺序，矩阵可能只有一行、只有一列，或者是更一般的长方形。答案应包含每个元素恰好一次。

螺旋遍历的本质是维护四个边界：`top`、`bottom`、`left`、`right`。每完成一圈，就把对应边界向内收缩一格。

每一轮依次输出上边、右边、下边、左边四条边。关键边界条件是：某一条边输出完后要立刻判断是否已经越界，因为当矩阵只剩一行或一列时，后面的三条边可能已经不存在。

只要边界更新与越界判断正确，就能保证每个元素恰好访问一次。

## 旋转图像

!!! problem "[48. 旋转图像](https://leetcode.cn/problems/rotate-image/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个 `n x n` 的二维矩阵 `matrix` 表示一个图像，请将图像顺时针旋转 `90` 度。旋转后的结果仍然是同样大小的方阵，只是元素位置发生了对应变化。

    题目要求原地修改矩阵，不额外申请同规模的辅助矩阵。换言之，需要在保持元素总数不变的前提下完成旋转。

方阵旋转可以拆成两个独立步骤：

1. 沿主对角线转置。
2. 再把每一行左右翻转。

转置会把 `matrix[i][j]` 和 `matrix[j][i]` 交换，相当于把行和列对调。随后每行反转一次，就把原本在左侧的元素移到右侧，完成顺时针旋转。

这种写法只需要原地交换元素，不需要额外矩阵，时间复杂度是 $O(n^2)$，空间复杂度是 $O(1)$。

## 搜索二维矩阵 II

!!! problem "[240. 搜索二维矩阵 II](https://leetcode.cn/problems/search-a-2d-matrix-ii/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个 `m x n` 矩阵 `matrix` 和目标值 `target`，请判断 `target` 是否出现在矩阵中，并返回布尔结果。矩阵满足每行从左到右升序、每列从上到下升序的性质。

    题目希望利用二维单调性进行高效查找，而不是对整个矩阵做完整遍历。只要找到了目标值就返回真，否则返回假。

矩阵同时满足“行单调”和“列单调”时，最适合从右上角开始搜索。

设当前位置是右上角元素 `x`：

- 如果 `x == target`，直接返回真。
- 如果 `x > target`，那么当前列下面的值都更大，整列都可以丢弃，因此向左移动。
- 如果 `x < target`，那么当前行左边的值都更小，整行都可以丢弃，因此向下移动。

每一步都能排除一整行或一整列，最多移动 `m + n` 次，时间复杂度为 $O(m+n)$。
