# 堆

堆分组里的题目通常不要求完整排序，而是要求在动态数据里快速拿到“前 k 个”或“中位数”这类局部有序结果。优先队列适合把选择问题降成增量维护问题。

## 数组中的第 K 个最大元素

!!! problem "[215. 数组中的第 K 个最大元素](https://leetcode.cn/problems/kth-largest-element-in-an-array/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。这里的“第 `k` 大”是按从大到小排序后的第 `k` 个位置来定义的。

    你只需要输出这个元素本身，而不需要返回排序后的整个数组。`k` 保证是合法范围内的正整数。

如果只关心第 `k` 大，而不关心完整排序结果，就没有必要维护一个完整的大根堆。更合适的做法是维护一个大小为 `k` 的小根堆。

遍历数组时：

- 堆的大小不足 `k` 时，直接入堆。
- 一旦堆大小超过 `k`，就弹出堆顶，也就是当前这 `k+1` 个元素里最小的那个。

遍历结束后，小根堆里保留下来的正好是最大的 `k` 个元素，其中堆顶就是第 `k` 大元素。这样时间复杂度是 $O(n \log k)$，空间复杂度是 $O(k)$。

## 前 K 个高频元素

!!! problem "[347. 前 K 个高频元素](https://leetcode.cn/problems/top-k-frequent-elements/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给你一个整数数组 `nums` 和一个整数 `k`，请返回其中出现频率前 `k` 高的元素。答案中的元素顺序不作要求，只要这 `k` 个高频元素都被找出来即可。

    如果多个元素的频率相同，只要它们属于频率最高的前 `k` 个候选集合，就都可以出现在结果中。题目关注的是选出高频元素集合，而不是完成完整排序。

这题可以拆成两步：先统计频率，再从频率表中选出前 `k` 个元素。统计频率通常用哈希表，选择阶段则适合小根堆。

设 `cnt[x]` 表示元素 `x` 的出现次数。把所有 `(cnt[x], x)` 放进大小为 `k` 的小根堆，堆顶始终保存当前候选里频率最小的元素。每次加入新元素后，如果堆大小超过 `k`，就弹出堆顶，保留更高频的部分。

最后堆中的元素就是答案集合。这个方法不需要对全部元素按频率完全排序，因此比直接排序更节省时间。

## 数据流的中位数

!!! problem "[295. 数据流的中位数](https://leetcode.cn/problems/find-median-from-data-stream/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    设计一个支持以下两种操作的数据结构：

    - `addNum(int num)`，从数据流中添加一个整数。
    - `findMedian()`，返回目前所有元素的中位数。

    中位数是有序数组中间位置的元素；如果元素个数是偶数，则返回中间两个元素的平均值。也就是说，`findMedian()` 需要随时反映当前已经加入的全部元素状态。

    该数据结构面向的是持续插入的在线场景，不能依赖每次查询前重新排序全部数据。

中位数问题的关键是同时维护“左半部分的最大值”和“右半部分的最小值”。最常见的结构是两个堆：

- 大根堆保存较小的一半元素。
- 小根堆保存较大的一半元素。

为了让中位数可以直接从堆顶读出，需要保持两个不变量：

- 两个堆的大小差不超过 `1`。
- 大根堆中的所有元素都不大于小根堆中的所有元素。

插入新数时，先放入某个堆，再把堆顶元素搬到另一边并重新平衡。这样 `findMedian` 只需查看堆顶即可：如果总数为奇数，取更长那一边的堆顶；如果总数为偶数，取两边堆顶的平均值。
