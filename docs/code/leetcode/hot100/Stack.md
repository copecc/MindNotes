# 栈

栈分组通常对应三类问题：括号匹配、嵌套结构解析、以及单调栈。它们都依赖“后进先出”来恢复最近一次仍然有效的状态。

## 有效的括号

!!! problem "[20. 有效的括号](https://leetcode.cn/problems/valid-parentheses/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个只包含 `'('`、`')'`、`'{'`、`'}'`、`'['` 和 `']'` 的字符串 `s`，请判断它是否是一个有效的括号串。所谓有效，是指每个左括号都必须被同类型的右括号按正确顺序闭合。

    判断时需要保证所有括号都能一一配对，并且不能出现右括号先于对应左括号出现的情况。最终如果还有未匹配的左括号残留，字符串也应判定为无效。

有效括号序列的判断条件是：每个右括号都必须匹配最近出现的、尚未被匹配的同类左括号。这个“最近未匹配”的约束正好对应栈。

遍历字符串时：

- 读到左括号就入栈。
- 读到右括号时，栈顶必须是对应的左括号；如果栈空或类型不匹配，直接判定无效。

遍历结束后，栈必须为空，才能说明所有括号都成功配对。该方法是典型的线性栈匹配。

## 最小栈

!!! problem "[155. 最小栈](https://leetcode.cn/problems/min-stack/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    设计一个支持 `push`、`pop`、`top` 和 `getMin` 操作的栈，其中 `getMin` 需要在常数时间内返回当前栈中的最小元素。普通栈操作的语义与标准栈一致。

    题目要求在频繁入栈、出栈的过程中，最小值信息也能同步维护，而不是每次查询时重新扫描整个栈。

问题的关键不在于普通栈，而在于“最小值必须随栈操作同步更新”。如果只保存一个栈，遇到弹栈时无法知道之前的最小值是什么。

常见做法是维护两个栈：

- 数据栈保存所有元素。
- 辅助栈保存“到当前为止的最小值”。

每次 `push` 时，如果新元素不大于当前最小值，就把它也压入辅助栈。每次 `pop` 时，如果弹出的元素正好等于辅助栈顶，也要同步弹出。这样 `getMin` 只需读取辅助栈顶即可，始终是 $O(1)$。

## 字符串解码

!!! problem "[394. 字符串解码](https://leetcode.cn/problems/decode-string/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个经过编码的字符串 `s`，请返回它解码后的结果。编码形式遵循 `k[encoded_string]`，表示方括号内的字符串需要重复 `k` 次后再拼接到结果中。

    这里的编码可能嵌套出现，例如一个方括号内部还包含更深层的编码片段。`k` 保证为正整数，解码时需要完整还原所有层级的结果。

这道题的难点在于嵌套结构：当遇到新的 `[` 时，必须先把外层上下文保存起来，等当前层 `]` 结束后再恢复外层状态。这个过程适合用栈保存分层信息。

通常维护两个栈：

- `cnt` 保存当前层的重复次数。
- `str` 保存进入这一层之前已经构造好的前缀。

当读到数字时，累积次数；读到 `[` 时，把当前前缀和次数入栈，并清空当前状态；读到 `]` 时，把当前片段重复 `k` 次，再和栈顶前缀拼接回去。这样可以自然处理多层嵌套。

## 每日温度

!!! problem "[739. 每日温度](https://leetcode.cn/problems/daily-temperatures/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个整数数组 `temperatures`，其中 `temperatures[i]` 表示第 `i` 天的温度，请返回数组 `answer`。`answer[i]` 表示从第 `i` 天开始，距离下一次出现更高温度还要经过多少天；如果之后再也没有更高温度，则 `answer[i] = 0`。

    输出数组的长度与输入相同，每个位置都要对应到“第一次更高温度”的等待天数，而不是任意一个更高温度出现的距离。

这是一道典型的单调栈题。栈中存放的是“还没有找到更高温度的下标”，并且这些下标对应的温度保持单调递减。

当遍历到新的温度 `temperatures[i]` 时，只要它比栈顶下标对应的温度更高，就说明栈顶那个位置第一次遇到了更高温度，答案可以立刻确定为 `i - idx`。随后继续弹栈，直到栈顶温度不再低于当前温度。

这种写法保证每个下标只会进栈和出栈一次，因此时间复杂度是 $O(n)$。

## 柱状图中最大的矩形

!!! problem "[84. 柱状图中最大的矩形](https://leetcode.cn/problems/largest-rectangle-in-histogram/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个由 `n` 个非负整数组成的数组 `heights`，其中每个数表示柱状图中一根柱子的高度，所有柱子的宽度都为 `1`。请计算在这个柱状图中能够勾勒出的最大矩形面积。

    矩形必须由连续的柱子组成，矩形的高度不能超过覆盖范围内最矮柱子的高度。返回值是能够取得的最大面积，而不是具体的矩形位置。

最大矩形的高度一定来自某根柱子，而它能向左、向右延伸的边界是“第一个比它矮的柱子”。因此问题可以转化为为每个柱子寻找左右两侧第一个更小元素。

单调递增栈可以一次性完成这件事。栈中保存柱子下标，并保持对应高度递增。遍历到更矮的柱子时，不断弹出栈顶：

- 被弹出的柱子就是当前以它为高的最大矩形的候选高度。
- 弹栈后新的栈顶就是它左侧第一个更小元素。
- 当前下标就是它右侧第一个更小元素。

有了左右边界，面积就能直接计算。末尾补一个高度为 `0` 的哨兵，可以强制把栈中剩余元素全部结算。
