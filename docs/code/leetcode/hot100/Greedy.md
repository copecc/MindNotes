# 贪心

贪心题的关键不是“看到局部最优就选”，而是要先把状态压缩成一个可以维护的不变量。Hot100 里的贪心题，常见的状态通常只有三类：当前最远可达位置、当前区间右端点、当前前缀最优值。

## 买卖股票的最佳时机

!!! problem "[121. 买卖股票的最佳时机](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个整数数组 `prices`，其中 `prices[i]` 表示第 `i` 天的股票价格。你只能选择某一天买入，并在后面的某一天卖出，且最多完成一次交易。请返回能够获得的最大利润。

    如果在任意时刻都无法获得正收益，则返回 `0`。题目保证买入必须发生在卖出之前，不能同时持有多次交易。

这题只允许一次买入和一次卖出，所以问题可以改写为：在每一天作为卖出日时，寻找它之前出现过的最低买入价。

扫描到第 `i` 天时，维护一个前缀最小值 `minPrice`，它表示前面所有天里最便宜的价格。此时如果在第 `i` 天卖出，利润就是：

$$
prices[i] - minPrice
$$

再用这个利润去更新答案即可。与此同时，`minPrice` 还要继续更新为当前前缀最小值。这样一次线性扫描就覆盖了所有可能的买卖组合。

这个贪心成立的原因很直接：卖出日固定时，最优买入日一定是此前价格最低的那一天；不需要保留更多历史信息。

## 跳跃游戏

!!! problem "[55. 跳跃游戏](https://leetcode.cn/problems/jump-game/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个非负整数数组 `nums`，你最初位于数组的第一个下标。`nums[i]` 表示你在下标 `i` 处最多可以向右跳跃的步数，实际跳跃长度可以小于等于 `nums[i]`。请判断是否能够到达最后一个下标。

    题目要求返回一个布尔值：能够到达时返回 `true`，否则返回 `false`。数组长度至少为 `1`，且起点始终是下标 `0`。

这题的核心是维护“当前已经能够到达的最远位置”。

设 `reach` 表示当前扫描前缀里，所有位置能够共同覆盖到的最远下标。遍历到位置 `i` 时：

- 如果 `i > reach`，说明连当前位置都到不了，后面就不可能继续前进，直接返回 `false`。
- 否则用 `i + nums[i]` 更新 `reach`，因为站在当前位置能把可达范围再向右扩展一段。

当 `reach` 覆盖到最后一个下标时，答案就成立。

这个过程本质上是在维护一个可达区间，而不是模拟具体跳法。只要某个位置已经在可达区间内，它提供的跳跃能力就可以合并进当前边界。

## 跳跃游戏 II

!!! problem "[45. 跳跃游戏 II](https://leetcode.cn/problems/jump-game-ii/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个非负整数数组 `nums`，你最初位于数组的第一个下标。`nums[i]` 表示你在下标 `i` 处最多可以向右跳跃的步数。请返回到达最后一个下标所需的最少跳跃次数。

    题目通常保证最后一个下标可以到达。每次跳跃都可以选择 `1` 到 `nums[i]` 之间的任意步数，目标是在所有可行方案中使跳跃次数最少。

这题不能只看“能否到达”，还要统计最少跳几次。贪心做法是把所有位置按“层”处理。

设：

- `curEnd` 表示当前这一步跳跃能够覆盖到的最右端。
- `farthest` 表示在当前这一层中，下一步能够扩展到的最远位置。

遍历数组时不断更新 `farthest = max(farthest, i + nums[i])`。当 `i` 走到 `curEnd` 时，说明当前这一跳能覆盖的所有位置都已经扫描完了，此时必须再跳一次，并把新的边界更新为 `farthest`。

这个过程等价于广度优先搜索按层扩展，但实现上只需要一次扫描。每一跳都选择当前层里能把下一层推得最远的覆盖范围，因此得到的跳数最少。

## 划分字母区间

!!! problem "[763. 划分字母区间](https://leetcode.cn/problems/partition-labels/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个只包含小写英文字母的字符串 `s`，请将它划分为尽可能多的片段，使得同一个字母只会出现在其中一个片段里。换句话说，每个字符在整个字符串中的所有出现位置都必须被包含在同一个片段中。

    返回划分后每个片段的长度，答案按从左到右的顺序输出。每个片段都必须是连续子串，并且所有片段拼接后要还原为原字符串。

这题的关键是先知道每个字符最后一次出现在哪里。

如果某个片段当前扫描到了位置 `i`，并且片段内所有字符的最远右端点的最大值就是 `right`，那么：

- 只要 `i < right`，这个片段就不能结束，因为里面至少有一个字符还会在后面出现。
- 当 `i == right` 时，说明当前片段里的所有字符都已经在这个区间内闭合，可以切分。

因此只要先预处理每个字符的最后位置，再从左到右扫描并动态维护当前片段的右端点即可。

这个做法的本质是区间合并：每个字符都会贡献一个必须覆盖的最右边界，最后把这些边界收缩成若干个互不交叠的闭区间。
