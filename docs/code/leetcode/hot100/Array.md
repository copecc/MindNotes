# 数组

数组分组里的题目主要围绕三类约束：区间如何合并、位置如何重排、以及前后缀信息如何在一次扫描中维护。它们的共同点是都依赖数组的连续存储特性，把“局部更新”转化成可重复利用的线性过程。

## 最大子数组和

!!! problem "[53. 最大子数组和](https://leetcode.cn/problems/maximum-subarray/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个整数数组 `nums`，请你找出一个具有最大和的连续子数组，并返回这个最大和。子数组必须保持原数组中的连续顺序，至少包含一个元素。

    题目要求返回的是最大子数组和的数值，而不是子数组本身。即使数组中全是负数，也要选出其中和最大的那个单元素子数组作为答案。

这道题的关键在于区分“以当前位置结尾的最优值”和“全局最优值”。

设 `dp[i]` 表示必须以 `nums[i]` 结尾的最大子数组和，那么只有两种选择：

$$
dp[i] = \max(nums[i],\ dp[i-1] + nums[i])
$$

如果前一段的贡献是负数，就应该丢弃前缀，从当前位置重新开始；如果前一段仍然为正，就把当前位置接在后面。这个转移只依赖前一个状态，因此可以压缩成一个滚动变量。

遍历过程中再维护一个全局最大值，就能得到答案。该做法只需要一次线性扫描，时间复杂度是 $O(n)$，额外空间复杂度是 $O(1)$。

## 合并区间

!!! problem "[56. 合并区间](https://leetcode.cn/problems/merge-intervals/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    以数组 `intervals` 表示若干区间，其中 `intervals[i] = [starti, endi]`。请合并所有有交集的区间，并返回一个不包含重叠区间的结果数组。区间的起止端点都需要保留在输出中。

    两个区间只要存在公共部分，就应该合并成一个更大的区间；如果两个区间完全分离，则需要在结果中分别保留。返回结果的顺序不作额外要求。

区间合并的前提是先把区间按左端点排序。排序后，所有可能重叠的区间一定会出现在相邻位置，因此只要维护当前已经合并出的最后一个区间 `last`，再顺序考察新区间 `cur` 即可。

若 `cur.start <= last.end`，说明二者有交集，新的右端点只需要取两者最大值：

$$
last.end = \max(last.end,\ cur.end)
$$

若 `cur.start > last.end`，说明当前区间与之前的结果完全分离，直接把 `cur` 作为新的合并段加入答案。

排序占用 $O(n \log n)$，线性扫描是 $O(n)$，整体复杂度由排序主导。

## 轮转数组

!!! problem "[189. 轮转数组](https://leetcode.cn/problems/rotate-array/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个数组 `nums` 和一个非负整数 `k`，请将数组中的元素向右轮转 `k` 个位置。轮转后，原来位于末尾的元素会依次移动到前面，元素相对顺序需要按轮转结果重新排列。

    题目要求在原数组上完成修改，不额外返回新的轮转数组。若 `k` 大于数组长度，实际移动次数应先对数组长度取模。

轮转操作要求原地完成，因此不能把结果拷贝到另一个数组再回填。常用做法是三次翻转：

1. 先把整个数组翻转。
2. 再翻转前 `k` 个元素。
3. 最后翻转剩余的 `n-k` 个元素。

设原数组为

$$
a_0,a_1,\dots,a_{n-k-1},a_{n-k},\dots,a_{n-1}
$$

整体翻转后变成

$$
a_{n-1},\dots,a_{n-k},a_{n-k-1},\dots,a_0
$$

再分别翻转两段，就能恢复出轮转后的顺序。这个过程只做若干次原地交换，时间复杂度是 $O(n)$，额外空间复杂度是 $O(1)$。

## 除自身以外数组的乘积

!!! problem "[238. 除自身以外数组的乘积](https://leetcode.cn/problems/product-of-array-except-self/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个整数数组 `nums`，请返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余所有元素的乘积。输出数组与输入数组长度相同，每个位置对应一个“去掉自己后”的乘积结果。

    题目保证答案在 32 位整数范围内，并且通常不允许使用除法来直接计算结果。数组中如果出现 `0`，也需要得到正确输出。

这道题不适合直接除法，因为数组里可能出现 `0`，而且题目通常也会限制不能使用除法。正确做法是把“除自身以外的乘积”拆成左右两侧的乘积之和。

对每个位置 `i`，答案可以写成：

$$
answer[i] = \left(\prod_{j < i} nums[j]\right)\left(\prod_{j > i} nums[j]\right)
$$

因此可以先从左到右维护前缀乘积，把左边贡献写进结果数组；再从右到左维护后缀乘积，把右边贡献乘回去。这样每个位置只被访问常数次，不需要额外数组保存全部前后缀。

这个做法自然覆盖 `0` 的情况，因为前缀和后缀是分开计算的，不依赖整体可逆性。

## 缺失的第一个正数

!!! problem "[41. 缺失的第一个正数](https://leetcode.cn/problems/first-missing-positive/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个未排序的整数数组 `nums`，请找出其中没有出现的最小正整数并返回。这里的正整数从 `1` 开始，答案一定是某个尚未出现在数组中的正数。

    题目要求算法在 `O(n)` 时间内完成，并且只使用常数级额外空间，因此不能依赖完整排序或额外的大规模辅助结构。

这道题的约束是“最小正整数”，因此只需要关心 `1` 到 `n` 之间的数，其中 `n = nums.size()`。如果某个值 `x` 落在这个范围内，那么它理想的位置就是下标 `x - 1`。

做法是原地交换：当 `nums[i]` 属于 `[1, n]` 且它还没有被放到正确位置时，就把它交换到 `nums[nums[i] - 1]`。这个过程会不断把当前值送回它应该在的位置，直到当前位置上的数不再满足可交换条件。

完成重排后，再从左到右扫描第一个不满足 `nums[i] = i + 1` 的位置，这个位置对应的正整数就是答案；如果全部匹配，答案就是 `n + 1`。

原地交换保证了每个数最多被放置一次，整体仍是 $O(n)$。
