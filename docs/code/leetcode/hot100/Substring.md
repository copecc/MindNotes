# 子串

这一页整理的是连续区间上的几种常见手法：前缀和配合哈希、单调队列，以及计数型覆盖窗口。它们都处理“连续”，但状态约束并不相同，不能简单地都归为普通滑动窗口。

## 和为 K 的子数组

!!! problem "[560. 和为 K 的子数组](https://leetcode.cn/problems/subarray-sum-equals-k/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给你一个整数数组 `nums` 和一个整数 `k`，请统计并返回数组中和为 `k` 的连续子数组个数。

    子数组必须是连续的一段区间。题目要求的是满足条件的区间数量，而不是区间本身。

这题不依赖窗口单调性，负数存在时，普通滑动窗口无法保证“收缩后和一定变小”。更合适的做法是前缀和加哈希计数。

设前缀和为：

$$
prefix[i] = nums[0] + nums[1] + \cdots + nums[i-1]
$$

那么区间 `[l, r]` 的和可以写成：

$$
prefix[r + 1] - prefix[l] = k
$$

等价于：

$$
prefix[l] = prefix[r + 1] - k
$$

因此在遍历到当前位置时，只需要知道“之前出现过多少次某个前缀和”，就能直接统计以当前右端点结尾、和为 `k` 的子数组数量。

## 滑动窗口最大值

!!! problem "[239. 滑动窗口最大值](https://leetcode.cn/problems/sliding-window-maximum/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给你一个整数数组 `nums` 和一个整数 `k`，请找出每个长度为 `k` 的连续子数组中的最大值，并按窗口从左到右的顺序返回这些最大值组成的数组。

    这里的输出是一个数组，数组中的第 `i` 项对应窗口 `[i, i + k - 1]` 的最大值。题目要求处理的是所有连续窗口，而不是只返回某一个窗口的结果。

这题的窗口是固定长度，但窗口内需要快速查询最大值。普通数组在每次移动时都重新扫描会退化成 $O(nk)$，因此需要用单调队列维护候选值。

队列中保存的是下标，并且下标对应的数值保持单调递减：

- 新元素进来时，把队尾所有不大于它的元素弹出。
- 窗口左边界右移时，把已经滑出窗口的队首元素弹出。
- 队首始终是当前窗口的最大值下标。

这样每个元素最多进队一次、出队一次，总复杂度是 $O(n)$。

## 最小覆盖子串

!!! problem "[76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给你两个字符串 `s` 和 `t`，请在 `s` 中找到一个包含 `t` 中所有字符的最短连续子串，并返回这个子串。如果不存在这样的子串，则返回空字符串 `""`。

    子串中需要覆盖 `t` 的每个字符及其出现次数，因此如果 `t` 中某个字符出现多次，窗口里也必须包含相同次数。

这题也是覆盖型窗口，但目标不是“找到一个满足条件的窗口”，而是“在满足条件的前提下尽量缩短窗口”。

维护一个计数数组 `need`，表示当前窗口距离满足 `t` 还差多少个字符。右端点扩张时，如果某个字符正好补上了缺口，就减少缺口计数；当缺口已经为 `0` 时，说明当前窗口已经覆盖 `t`。

一旦窗口满足条件，就不断移动左端点尝试收缩：

- 收缩前先更新答案。
- 如果移出左端点后窗口重新失去某个必需字符，就停止收缩，继续扩张右端点。

因此这题的关键状态不是“窗口是否合法”这么简单，而是“合法后如何继续最小化长度”。
