# 双指针

双指针题的关键不是“同时维护两个下标”，而是让两个下标分别承担不同的不变量。常见场景包括相向收缩、同向扫描，以及固定一端后在剩余区间内继续搜索。

## 盛最多水的容器

!!! problem "[11. 盛最多水的容器](https://leetcode.cn/problems/container-with-most-water/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个长度为 `n` 的非负整数数组 `height`，其中每个元素表示坐标轴上的一条竖直线高度。请从中选择两条线，使它们与 `x` 轴共同围成的容器能够盛放最多的水，并返回这个最大面积。

    容器的宽度由两条线之间的距离决定，高度由较短的那条线决定。也就是说，只有同时考虑宽度和有效高度，才能得到最终答案。

容器面积由两侧边界的较短边决定：

$$
area(i, j) = \min(height_i, height_j) \times (j - i)
$$

初始时让指针位于两端 `left = 0`、`right = n - 1`。此时宽度最大，面积上限已经固定；后续想让面积继续变大，唯一可能是让较短的一侧变高。若移动较高的一侧，宽度只会缩小，而短边不变，面积不会获得更有利的变化。

因此每一步都移动当前较短的一侧：

- 若 `height[left] < height[right]`，向右移动 `left`。
- 否则向左移动 `right`。

这个策略保证每个状态只保留可能产生更优答案的一侧，时间复杂度为 $O(n)$。

## 三数之和

!!! problem "[15. 三数之和](https://leetcode.cn/problems/3sum/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给你一个整数数组 `nums`，请找出所有满足 `a + b + c = 0` 的三元组 `[a, b, c]`。返回结果中不能包含重复的三元组，顺序可以任意。

    数组中可能存在重复元素，因此同一个数值组合只能记录一次。题目要求返回的是所有满足条件且互不重复的解，而不是判断是否存在解。

这题的核心是把“三个数求和”转成“固定一个数后，在有序区间里找两数之和”。

先对数组排序。排序的作用有两层：

- 便于用左右指针在剩余区间内线性搜索。
- 便于跳过重复值，避免输出重复三元组。

设当前固定的数是 `nums[i]`，目标就变成在区间 `[i + 1, n - 1]` 中寻找两数之和 `-nums[i]`。若当前和过小，就增加左指针；若当前和过大，就减少右指针。由于区间已经有序，指针移动方向是唯一确定的。

去重要分三处处理：

- 固定数 `nums[i]` 需要跳过和前一个相同的值。
- 找到一组解后，左指针要跳过连续重复值。
- 找到一组解后，右指针也要跳过连续重复值。

这样可以保证每个三元组只会被记录一次。

## 移动零

!!! problem "[283. 移动零](https://leetcode.cn/problems/move-zeroes/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定一个整数数组 `nums`，请将所有 `0` 移动到数组末尾，同时保持所有非零元素的相对顺序不变。

    题目要求在原数组上完成这个调整，返回结果时数组本身应已经被修改。也就是说，不能通过重新构造一个新数组来规避原地处理的要求。

这题使用同向双指针：

- `scan` 负责从左到右遍历全部元素。
- `write` 记录下一个非零元素应该放入的位置。

遍历过程中，`scan` 之前已经看过的部分中，所有非零元素都会被依次放到 `write` 指向的位置。这样得到的是一个稳定的原地分区：非零元素保持原相对顺序，零被自然挤到后面。

之所以可以直接交换，是因为 `write` 指向的位置要么已经是当前元素本身，要么是一个需要被后续零覆盖的位置。整个过程只扫描一遍，时间复杂度是 $O(n)$。

## 接雨水

!!! problem "[42. 接雨水](https://leetcode.cn/problems/trapping-rain-water/description/?envType=study-plan-v2&envId=top-100-liked){target=_blank}"

    给定 `n` 个非负整数，表示每个宽度为 `1` 的柱子的高度图。请计算按照这些柱子排列之后，下雨时能够接到的总雨水量。

    每根柱子的宽度相同，雨水只能积在相邻柱子形成的凹槽中。最终返回的是所有位置累计得到的水量。

这一题的面积不是由两个端点直接决定，而是由当前位置左侧最高挡板和右侧最高挡板共同决定：

$$
water[i] = \min(leftMax[i], rightMax[i]) - height[i]
$$

双指针做法不需要显式预处理两个数组，而是同步维护两侧的当前最高值。

若 `height[left] < height[right]`，那么当前位置左侧的短板已经确定，当前位置能接的水只由 `leftMax` 决定；右侧再怎么提高，也不会改变当前左侧位置的上界。因此可以安全地处理 `left` 并向右移动。

对称地，若 `height[left] \ge height[right]`，就处理 `right` 并向左移动。

这个过程维持的核心不变量是：较短一侧的位置，其能接的水量已经可以被当前侧的最大值确定。这样每个位置只会被处理一次。
