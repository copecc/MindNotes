# 推理时扩展

!!! abstract

    本文整理 inference-time scaling 的主要方法，重点说明 best-of-N、self-consistency、verifier-guided decoding 与搜索分别如何利用额外推理计算提升结果质量，以及它们与后训练之间的关系。这里关注的是在线阶段如何把已有能力转化为更高成功率，而不是把所有性能提升都归因于参数规模增长。

## 基本思想

inference-time scaling 的核心，是在模型参数固定的前提下，增加采样、评估或搜索计算，以提高最终输出质量。与训练扩展不同，它不改变模型权重，只改变部署时愿意为单个请求投入多少计算预算。

其收益来源通常有两类。

- 候选增广：让模型生成更多可能解
- 选择增广：用投票、verifier 或搜索规则从更多候选中挑出更优结果

因此，inference-time scaling 的前提不是“计算越多越好”，而是候选分布中确实存在可被额外计算挖出的更优答案。

## Best-of-N

best-of-N 是最直接的方法：对同一输入采样 $N$ 个候选，再依据某种评分规则选出最优结果。

若单次采样命中高质量答案的概率为 $p$，则至少命中一次的概率为：

$$
1 - (1 - p)^N.
$$

这个公式解释了为什么 best-of-N 往往在 reasoning 和代码任务上有效：只要模型偶尔能答对，重复采样就可能放大成功率。

best-of-N 的优点是实现简单、与底层模型解耦。其代价也很明确。

- 成本近似随 $N$ 线性增长
- 收益依赖评分器是否能识别真正更好的候选
- 当候选高度同质化时，额外样本带来的边际收益会下降

## Self-Consistency

self-consistency 通常用于多步推理任务。系统先采样多条推理路径，再对最终答案做聚合，例如多数投票。其核心假设是：正确答案往往能被多条不同合理路径重复支持，而错误路径的终点更分散。

与普通 best-of-N 相比，self-consistency 更强调答案层面的聚合，而不是只对整条文本打分。它的优势在于无需单独训练 verifier，也能在部分任务上稳定提升准确率。

它的边界同样明显。

- 当错误模式高度相关时，多数投票会共同偏向同一错误答案
- 当答案空间连续或开放时，投票聚合会变得困难
- 当采样温度过低时，不同轨迹之间缺乏足够多样性

因此，self-consistency 的关键不只是采样次数，还包括候选独立性与答案聚合方式。

## Verifier-Guided Decoding

verifier-guided decoding 在生成过程中或生成后引入外部评分器，用于选择更优 token、局部步骤或整条轨迹。它比 self-consistency 更主动，因为系统不只是做事后投票，还可以在搜索过程中持续接收引导信号。

常见形式包括：

- 对完整候选打分后重排
- 对中间步骤做局部评分并提前剪枝
- 在 beam search 或 tree search 中把 verifier 分数并入路径优先级

其优势是能利用更细粒度的质量信号。其代价是系统复杂度更高，对 verifier 质量更敏感。若 verifier 与真实目标错位，模型会优先优化被打高分的代理特征，而非真实答案质量。

## 搜索

更一般的 inference-time scaling 可以视为搜索问题。状态是当前上下文或部分解，动作是下一步 token、子结论或工具调用，评价函数来自模型概率、verifier、规则或执行结果。

常见搜索形式包括：

| 方法 | 核心机制 | 优势 | 代价 |
| --- | --- | --- | --- |
| Beam Search | 保留前若干高分路径 | 稳定、实现成熟 | 容易缺少多样性 |
| Tree Search | 显式展开分支并回溯 | 适合复杂推理与规划 | 状态爆炸明显 |
| MCTS 风格搜索 | 平衡探索与利用 | 更适合长程决策 | 需要精细设计 rollout 与 value |
| ReAct 式多轮搜索 | 推理与动作交替 | 适合工具环境 | 环境延迟高，错误链更长 |

搜索的收益不仅取决于深度或宽度，还取决于剪枝规则是否有效。若没有足够强的价值信号，搜索很容易把计算花在低质量分支上。

## 与后训练的关系

inference-time scaling 与后训练是互补关系。

- 后训练提高候选分布质量，使高质量答案更容易被采到
- 推理时扩展提高选择质量，使已有好候选更容易被保留下来

这可以理解为“生成上限”和“选择效率”的分工。若模型从未学会某类推理模式，搜索空间中就缺乏有效候选；若模型已经偶尔能答对，推理时扩展通常能较低风险地提高成功率。

从工程角度看，是否采用 inference-time scaling，取决于离线训练成本和在线延迟预算如何分配。高并发低延迟服务更倾向单次解码或小规模 rerank；高价值低频任务则更适合更深的搜索或多样本验证。

## 选择边界

是否值得使用 inference-time scaling，通常取决于以下条件。

- 单次解码是否已经接近上限
- 候选之间是否具有足够多样性
- 是否存在可靠评分器或可执行验证环境
- 任务价值是否足以覆盖额外延迟与成本

如果这些条件同时较弱，额外计算常常只能放大噪声而不是放大能力。

## 例子

### 数学推理

对可判分数学题，可以采样多条链式推理，用答案检查器或多数投票选出最可信结果。此处 self-consistency 与 best-of-N 都较容易生效。

### 代码生成

对代码任务，可以生成多个候选并运行单元测试，测试结果相当于强 verifier。此时 inference-time scaling 的收益通常高于开放式文本任务，因为评价信号更明确。

### 工具代理

对需要外部环境交互的代理任务，系统可以在每一步维护若干计划分支，并依据 observation 与 verifier 进行剪枝。这里的主要限制来自环境延迟与长轨迹误差积累。

## 要点

- inference-time scaling 通过增加在线计算提升输出质量
- best-of-N 放大命中率，self-consistency 放大答案聚合稳定性
- verifier-guided decoding 把外部评价信号引入选择过程
- 搜索收益依赖候选质量、评分器质量与剪枝效率
- 后训练决定候选上限，推理时扩展决定能否找到这些候选
